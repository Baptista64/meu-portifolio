SISTEMA DE GESTÃO ESCOLAR

O SISGER (de nome Julieta) é um sistema de gestão escolar que visa facilitar os
processos existentes em uma escola, focando preferencialmente nos processos
envolvendo os dados dos alunos, desde a organização, manutenção dos mesmos de
uma forma simples, prática e sobretudo, rápida.

Principais funcionalidades

1. Gerenciamento de dados dos alunos: o sistema permite o gerenciamento dos dados dos alunos, desde o cadastro do aluno no sistema, atualização dos dados, remoção dos dados do aluno no sistema e controle dos mesmos;
2. Gerenciamento de usuários: o sistema permite que se crie usuários com perfis específicos, alteração dos dados, eliminação dos dados e procura pelo dados de funcionário da instituição;
3. Restrições de acesso: o sistema limita o acesso das funcionalidades do sistema a partir dos perfis dos usuários, o que quer dizer, que alguns usuários não possuem acesso à algumas funcionalidades;
4. Acesso ao sistema: para que se faça qualquer atividade no sistema, é
necessário que o usuário faz primeiro o login por meio de seu perfil, ou seja, o usuário precisa de uma senha e de um “nome” para poder ter acesso ao sistema;
5. Gerenciamento de turmas: o sistema permite o controle dos dados de uma turma, dados como, o número de alunos presentes na turma, a classe em que a turma pertence, a sala, entre outros dados;
6. Gerenciamento de cursos: o sistema também permite que se crie cursos e se faça o controle e organização dos cursos;
7. Gerenciamento de disciplinas: o sistema permite que se gerencie os dados de uma disciplina, entre outras coisas, esse gerenciamento inclui, definir qual professor leciona uma disciplina exata, quais classes possuem essa disciplina e quais cursos;
8. Controle de notas: o sistema permite que se faça o controle das notas dos
alunos, permitindo que apenas o professor responsável por lecionar a disciplina, seja capaz de alterar as notas;
9. Limite de tempo de lançamento de notas: para um maior controle, pode-se limitar o prazo de lançamento de notas, definindo quando os professores podem ou não mudar e/ou adicionar notas aos alunos;
10. Inscrição de novos alunos: o sistema permite que se faça a inscrição de novos alunos, controlando as suas notas sem que estes sejam considerados
necessariamente alunos da instituição, ou seja, é possível cadastrar um inscrito e só depois colocá-lo como aluno da instituição caso a instituição considere que ele esteja apto para tal(apenas se aluno for aprovado em teste, por exemplo);
11. Confirmação de matrícula: o sistema permite que se faça a confirmação de matrícula tanto para alunos que foram aprovados, tanto para alunos que não foram aprovados;
12. Impressão de documentos: o sistema permite a impressão de documentos, como, emissão de cartão de estudante, comprovativo de pagamento de certificados, comprovativo de pagamento de propinas, impressão dos próprios certificados e declarações de maneira fácil e prática, sem a necessidade de se preencher os dados do aluno, desde que este esteja cadastrado no sistema;
13. Pagamento: o sistema permite o gerenciamento dos pagamentos de
documentos e também de propinas, permitindo a impressão do comprovativo no processo;
14. Busca dos dados de um pagamento: o sistema permite que se busque e imprima o recibo de algum pagamento caso o aluno o tenha perdido;
15. Publicação de eventos/notícias: o sistema permite que se publique algum aviso ou notícia para que os professores possam ter acesso a informação sem a necessidade de interação constante;
16. Registro de actividades: o sistema grava e permite verificar o que os usuários fazem no sistema, por exemplo, se um usuário alterar uma nota, o sistema grava a informação de quando a nota foi alterada se essa nota pertence a que aluno e qual usuário fez essa alteração;
17. Estatísticas: o sistema permite verificar as estatísticas da escola, algumas dessas estatísticas são, o número total de alunos, o número de turmas;
18. Cópias de segurança: o sistema permite que se faça cópias de segurança dos dados para que esses possam ser restaurados em caso de dano ou perca de dados por parte do servidor.
